# Wkład w SpiralMind Nexus

Dziękujemy za zainteresowanie rozwojem projektu! Prosimy o przestrzeganie poniższych zasad:

## Zgłaszanie błędów i propozycji
- Używaj szablonów zgłoszeń w zakładce Issues.
- Opisuj dokładnie problem lub propozycję.

## Pull Requesty
- Przed PR uruchom testy (`pytest`) i sprawdź pokrycie (`pytest --cov`).
- Przestrzegaj stylu kodu (flake8, black, mypy).
- Opisz zmiany w szablonie PR.
- Każda nowa funkcja powinna mieć testy.

## Styl kodu
- Python: PEP8, typowanie, docstringi.
- JS/TS: ESLint, Prettier.
